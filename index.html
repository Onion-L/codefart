CodeFart plays a sound when Claude Code finishes responding. Built on native hooks. Zero wrapping, zero trust issues. One command to install.

curl -fsSL https://raw.githubusercontent.com/Onion-L/codefart/main/install.sh | bash

Then run codefart setup to enable Claude notifications.

How it works

No wrappers. No aliases.
No trust issues.

CodeFart uses Claude Code's built-in Stop hook. Claude itself calls codefart — we never touch your process, your stdin, or your stdout.

Why CodeFart

Every other notification tool is boring. This one makes you smile — and you'll actually tell your friends about it.

⚡

One command install

curl | bash. That's it. Then codefart setup to enable Claude notifications. Under 60 seconds.

🔊

Real fart sounds

classic and wet themes built in. Add your own with codefart set-sound. Preview before you commit.

🛡️

Zero trust issues

No process wrapping. No alias trickery. Claude's own hook system calls codefart. Check ~/.claude/settings.json yourself.

🔄

Self-updating

codefart update pulls the latest release. Auto-elevates with sudo when needed. Never think about it again.

🎨

Interactive theme picker

Arrow keys to browse themes, Enter to select. Preview mode lets you audition without switching.

📦

Universal wrapper

codefart run -- npm run build. Wrap any command. Fart when it's done. Stupidly practical.

Usage

Commands you'll actually use.

Every action is a single command. No config files to edit.

codefart setupEnable Claude notifications (once)
codefart playPlay current sound
codefart listList themes, show current
codefart themeInteractive theme picker ↑↓ Enter
codefart theme classicSwitch directly
codefart previewAudition themes without switching
codefart set-sound ~/boom.mp3Use custom audio file
codefart removeRemove custom sound
codefart resetReset to defaults
codefart updateSelf-update to latest release
codefart run -- npm run buildWrap any command
